# Устранение проблем с загрузкой проекта и удаленным управлением

В случае, если при работе с iRidi Pro или i3 KNX сервер зависает при попытке загрузки проекта из облака или отсутствует удаленное управление, это может указывать на проблемы с сетевой доступностью европейских узлов облака (*.eu.iridi.com).

## Рекомендуемое решение: Обновление ПО

Установка актуальной сборки с функцией тестера хостов позволяет автоматически выбирать доступные серверы.

### Шаги по обновлению:
1. **Определите платформу сервера** (HSS, UMS или PC/Debian/Windows).
2. **Выберите соответствующий файл** по ссылкам в официальной документации или через поддержку.
3. **Установите обновление**:
   * **Через Web-интерфейс**: Раздел «Обновление» → «Ручное обновление».
   * **Через SSH**: Остановите службу (`/opt/iridiumserver/forebear.sh --stop`), скачайте файл `.ipk` и выполните `opkg install --force-reinstall <имя_файла.ipk>`.
4. **Перезагрузите сервер**.

## Временное решение (блокировка недоступных хостов)

Если обновление невозможно, выполните следующие действия в терминале SSH для принудительного игнорирования проблемных адресов:

```bash
echo "127.0.0.1 auth.eu.iridi.com" >> /etc/hosts
echo "127.0.0.1 i3pro.eu.iridi.com" >> /etc/hosts
echo "127.0.0.1 i3knx.eu.iridi.com" >> /etc/hosts
echo "127.0.0.1 bus77home.eu.iridi.com" >> /etc/hosts
rm /var/lib/iRidium\ Server/Documents/irhosts.json
reboot
```

*Примечание: После выполнения данных команд сервер перестанет обращаться к указанным доменам, что устранит зависание при загрузке.*
