В iRidi Pro предусмотрено несколько механизмов защиты проекта: от ограничения доступа конечных пользователей к сервисным страницам до защиты программного кода (скриптов) от просмотра и редактирования другими интеграторами.

### 1. Защита скриптов (JavaScript)

Это основной способ защиты логики вашего проекта. Вы можете установить пароль на любой скриптовый файл, что предотвратит его открытие в редакторе iRidi Studio без знания ключа.

*   **Как настроить:** В редакторе скриптов выберите нужный файл и воспользуйтесь функцией установки пароля (Password). 
*   **Результат:** При попытке открыть скрипт система запросит пароль. Скрипт будет исполняться в проекте, но его содержимое будет скрыто.

### 2. Ограничение доступа к страницам и попапам (Runtime)

Этот метод используется для того, чтобы запретить конечному пользователю или посторонним лицам открывать определенные части интерфейса в приложении.

#### Настройка через системные пароли:
1.  В **System Settings** проекта задайте пароли для уровней доступа (от 1 до 4).
2.  В настройках графического элемента (например, кнопки), который открывает страницу или попап, выберите соответствующий номер пароля (Password ID).
3.  При нажатии на кнопку в приложении появится системное окно ввода пароля.

#### Настройка через JavaScript API:
Для более гибкого управления (например, при наступлении события или вводе пароля в кастомное поле) используйте метод `IR.ValidatePassword()`.

```javascript
// Пример проверки пароля уровня 1
var confirm = IR.ValidatePassword(1, "1234"); // возвращает true, если пароль верен
if (confirm) {
    IR.Log("Доступ разрешен");
    // Выполните действие, например: IR.ShowPopup("Settings");
} else {
    IR.Log("Доступ запрещен");
}
```
*Важно: Метод работает асинхронно, поэтому логику, зависящую от результата, следует размещать внутри callback-функции.*

### 3. Защита файла проекта

Вы можете установить пароль на весь файл проекта в меню **Project Settings**. Это ограничит возможность открытия проекта в iRidi Studio для любого, кто не владеет паролем.

### 4. Рекомендации по защите авторских решений (Addons)

Если вы разрабатываете уникальные графические и программные модули («аддоны») для продажи или передачи другим интеграторам, учитывайте следующие особенности текущей архитектуры iRidi Pro:

1.  **Ограничение на защиту UI:** На данный момент iRidi Pro не поддерживает установку пароля на отдельные страницы или графические элементы внутри редактора (Design-time). Если интегратор получил доступ к файлу проекта, он сможет увидеть структуру страниц.
2.  **Динамическая генерация UI:** Для максимальной защиты рекомендуется создавать интерфейс (кнопки, попапы, индикаторы) динамически с помощью JavaScript. 
    *   Код создания интерфейса размещается в скрипте.
    *   Скрипт защищается паролем.
    *   В этом случае другой специалист не увидит элементы в дереве проекта, так как они создаются только в момент запуска приложения.
3.  **Лицензирование:** Для защиты коммерческих решений рекомендуется связывать работу скрипта с уникальным ID сервера или панели (Serial Number), чтобы ваш модуль нельзя было просто скопировать в другой проект без вашего разрешения.
