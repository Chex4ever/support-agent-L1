# Устранение проблем с отображением веб-интерфейса эмулятора iRidi

При работе с эмулятором iRidi Server в среде iRidi Studio могут возникнуть ситуации, когда веб-интерфейс сервера загружается некорректно, отображает пустые страницы или показывает аномально высокое потребление системных ресурсов (например, ОЗУ свыше 1000%).

### Признаки проблемы
* Невозможность загрузки веб-панели управления эмулятора.
* Ошибки при считывании данных о памяти и диске в интерфейсе.
* Потеря управления тегами.
* Интерфейс «ломается» или зависает на этапе инициализации.

### Причина возникновения
Основной причиной данного поведения является работа антивирусного ПО (в частности, **Kaspersky Endpoint Security**). Антивирус внедряет служебные скрипты в HTTP-трафик для проверки соединений. Это нарушает работу компонентов **Polymer/Web Components**, на которых построен интерфейс iRidi, даже если функции сетевого контроля формально отключены в настройках антивируса.

---

### Решение проблемы

#### 1. Добавление браузера в доверенную зону
Самым эффективным решением является добавление используемого вами браузера в список доверенных приложений антивируса:
1. Откройте панель управления Kaspersky.
2. Перейдите в раздел **«Настройка»** -> **«Дополнительно»** -> **«Сеть»**.
3. В блоке «Проверка защищённых соединений» вместо «Всегда проверять» выберите **«Проверять по запросу компонентов защиты»**.
4. Перейдите в раздел **«Доверенная зона»** -> **«Приложения»**.
5. Добавьте исполняемый файл вашего основного браузера (например, `chrome.exe`, `msedge.exe`) в список исключений.

#### 2. Очистка временных данных
Если проблема сохраняется, выполните очистку рабочей папки проекта:
1. Закройте iRidi Studio и эмулятор.
2. Перейдите по пути: `C:\Users\<Имя_пользователя>\Documents\iRidium pro documents`.
3. Удалите содержимое папки.
4. Запустите iRidi Studio от **имени администратора**.

#### 3. Настройка сетевого экрана (Firewall)
Убедитесь, что порты и приложения iRidi добавлены в исключения брандмауэра Windows и сетевого экрана антивируса:
* Добавьте в исключения папку: `%UserProfile%\Documents\iRidium pro documents\Server`.
* Разрешите сетевую активность для: `iRidiumServer32.exe` и `iRidiumServerRu32.exe`.
* Проверьте доступность локального адреса `127.0.0.1` и порта `8888` (по умолчанию).

---

> **Примечание:** Если вышеуказанные меры не помогли, рекомендуется временно приостановить защиту антивируса для подтверждения влияния ПО на работу эмулятора и обратиться в техническую поддержку производителя антивирусного решения с описанием проблемы внедрения скриптов в локальный трафик.
