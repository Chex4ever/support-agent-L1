# Настройка межсетевой маршрутизации и Multicast для iRidi RVR

Для обеспечения корректной работы сервера iRidi RVR (ReView Rooms) в сложных сетевых инфраструктурах, где оборудование и сервер находятся в разных подсетях (VLAN), необходимо выполнить специфические настройки маршрутизации и мультикаст-вещания.

### Физическое подключение и выбор интерфейса

Сервер iRidi RVR оснащен двумя LAN-портами. Для корректной работы рекомендуется следующая схема подключения:
1. **LAN 1**: Подключение к корпоративной сети (для управления и внешнего доступа).
2. **LAN 2**: Подключение к изолированной сети с оборудованием (автоматика, панели, контроллеры).

**Программная настройка:**
* В веб-интерфейсе ReView Rooms перейдите в раздел **Settings** -> **System**.
* В поле **Outbound interface** выберите сетевой интерфейс, который физически подключен к сети с оборудованием.
* **Важно:** После изменения Outbound interface и сохранения настроек необходимо перезагрузить сервер для вступления изменений в силу.

### Сетевые протоколы и порты

Для обнаружения устройств (Discovery) и мониторинга их состояния iRidi RVR использует мультикаст-трафик. 

**Используемые протоколы:**
* **UDP/Multicast**: Порт `30583` (0x7777). Используется для широковещательного поиска устройств, контроля статуса (Online/Offline) и сбора метрик.
* **TCP**: Используется для визуализации интерфейсов на ПК оператора, доступа к веб-интерфейсам контроллеров и системного взаимодействия.
* **PIM-SM / IGMP**: Необходимы для трансляции мультикаст-трафика между VLAN.

### Настройка сетевого оборудования (L3)

Для обеспечения прохождения трафика поиска (Discovery) между разными VLAN на уровне L3-коммутаторов или маршрутизаторов должны быть выполнены следующие настройки:

1.  **Глобальная поддержка Multicast**: Включите мультикаст-маршрутизацию на устройстве.
2.  **PIM-SM (Protocol Independent Multicast - Sparse Mode)**: Должен быть активирован на всех L3-интерфейсах (SVI), участвующих в обмене данными.
3.  **Rendezvous Point (RP)**: Назначьте основной маршрутизатор в качестве статической точки рандеву (Static RP) для управления группами мультикаста.
4.  **IGMP**: Включите поддержку IGMP на всех пользовательских интерфейсах (VLAN), где находятся сервер RVR и конечное оборудование.

### Особенности работы в многосегментных сетях

При настройке межсетевого обнаружения следует учитывать следующие аспекты:
* **Автоматический поиск (Scan)**: Корректно работает в пределах одной L2-сети. При работе через L3 (разные VLAN) функционал автоматического сканирования может быть ограничен спецификой сетевого оборудования.
* **Ручное добавление (Find device manually)**: Если автоматический поиск не находит устройство в другом VLAN, используйте ручное добавление по IP-адресу. 
* **Статус устройств**: Если после добавления устройство отображается как "Offline", убедитесь, что мультикаст-трафик по порту `30583` беспрепятственно проходит между VLAN сервера (Outbound interface) и VLAN оборудования.
