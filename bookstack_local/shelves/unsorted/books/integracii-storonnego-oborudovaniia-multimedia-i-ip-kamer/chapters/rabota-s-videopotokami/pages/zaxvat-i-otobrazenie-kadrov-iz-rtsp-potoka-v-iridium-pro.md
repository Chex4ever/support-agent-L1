# Захват и отображение кадров из RTSP-потока

В текущих версиях iRidium Pro прямая экстракция отдельных кадров из активного видеопотока RTSP не поддерживается. Для реализации функционала захвата снимков (snapshots) рекомендуется использовать один из двух подходов.

## Способ 1: Использование Snapshot URL камеры
Если ваша IP-камера поддерживает генерацию статических снимков по HTTP-запросу, это наиболее эффективный метод. 
* Настройте периодический запрос к URL снимка камеры.
* Полученные данные передавайте в графический элемент проекта.

Подробнее об управлении камерами можно узнать в [документации по PTZ-контролю](https://dev.iridi.com/IP_Cameras/en#PTZ_Control).

## Способ 2: Динамическое обновление через Base64
Если камера не имеет Snapshot URL, необходимо организовать внешнюю обработку потока и передачу данных в iRidium через метод `IR.CreateImageFromBuffer`.

### Алгоритм работы
1. **Внешний сервис:** Сторонний скрипт (Node.js, Python) захватывает кадр из RTSP (например, через `ffmpeg`), кодирует его в Base64.
2. **Передача данных:** Строка передается в iRidium через любой доступный драйвер (MQTT, HTTP, WebSocket или TCP).
3. **Отрисовка в проекте:** Скрипт панели принимает данные и обновляет изображение в памяти.

### Пример реализации (JS)
```javascript
IR.AddListener(IR.EVENT_ITEM_RECEIVE_DATA, IR.GetDevice("DataDriver"), function(text) {
    // Создание изображения в оперативной памяти (перезаписывается при каждом вызове)
    IR.CreateImageFromBuffer("LiveSnap", text);
    
    // Привязка созданного изображения к элементу интерфейса
    IR.GetItem("Page 1").GetItem("CamBox").GetState(0).Image = IR.GetImage("LiveSnap");
});
```

## Важные рекомендации и ограничения

### Оптимизация производительности
* **Объем данных:** Кодирование в Base64 увеличивает размер данных примерно на 33%. 
* **Частота обновления:** Для сохранения стабильности системы рекомендуется ограничить частоту кадров до **1–2 FPS**.
* **Сжатие:** Предварительно сжимайте изображение в JPEG на стороне сервера перед отправкой в iRidium.

### Оборудование iRidi (HSS)
Использование серверного оборудования iRidi (например, Home Server) для декодирования видеопотока сторонними утилитами (ffmpeg через SSH) **не рекомендуется**:
* Устройства ориентированы на задачи автоматизации, а не на видеопроцессинг.
* В ОС отсутствует менеджер пакетов для установки зависимостей.
* Стороннее ПО может привести к нестабильности системы и аннулированию гарантийных обязательств.

**Рекомендация:** Выносите задачи обработки видео на выделенный сторонний сервер или микрокомпьютер.
