# Интеграция iRidi Pro Server с iRidi SCADA Server

Интеграция системы автоматизации на базе iRidi Pro Server или iRidi HS Server с системой диспетчеризации iRidi SCADA Server позволяет объединить локальное управление инженерными системами и централизованный мониторинг на стороне диспетчера.

Такой подход может использоваться на объектах, где требуется разделить пользовательское управление и эксплуатационную диспетчеризацию. Например, iRidi Pro Server управляет оборудованием и выполняет основную логику автоматизации, а iRidi SCADA Server получает необходимые данные для мониторинга, отображения состояний и, при необходимости, передачи команд управления.

## Архитектурная концепция

В рекомендуемой архитектуре iRidi Pro Server или iRidi HS Server выступает основной точкой входа в систему автоматизации. Он взаимодействует с инженерным оборудованием, обрабатывает состояния, выполняет сценарии, расписания и другую проектную логику.

iRidi SCADA Server подключается к серверному проекту iRidi Server по IP-сети и получает из него необходимые теги. Эти теги могут использоваться для отображения состояний, контроля аварий, мониторинга параметров и передачи команд управления.

## Кейс: Интеграция вентиляционного оборудования

На практике часто возникает вопрос о том, как наиболее эффективно организовать диспетчеризацию стороннего оборудования (например, вентиляционных установок), уже подключенного к iRidi HS Server через шлюзы (KNX/Modbus).

**Важно:** Не рекомендуется использовать физические разветвители интерфейсов (например, ModBus RTU) для параллельного подключения к SCADA. Это повышает риск конфликтов на шине и усложняет диагностику.

**Рекомендуемое решение:**
1. Сохранить управление вентиляцией через iRidi HS Server.
2. Реализовать интеграцию через серверный проект: добавить iRidi Server в SCADA-проект.
3. Передать критически важные сигналы (состояние, режим работы, скорость вентилятора, уставки, фактические значения, аварии, статус связи) в SCADA как теги.
4. Вывести полученные данные на АРМ диспетчера для мониторинга и управления.

Такой подход позволяет SCADA работать с уже подготовленными данными, сохраняя единую логику управления на стороне основного сервера.

## Рекомендуемая схема интеграции

Рекомендуемая схема работы:

1. iRidi Pro Server или iRidi HS Server управляет инженерным оборудованием.
2. В серверном проекте формируются необходимые состояния, команды и уставки.
3. iRidi SCADA Server подключается к iRidi Server по IP-сети.
4. В SCADA-проект передаются необходимые теги из серверного проекта.
5. На стороне SCADA Client / АРМ диспетчера эти теги используются для мониторинга и управления.

## Преимущества схемы «Сервер — Сервер»

### 1. Единая точка управления
Управление оборудованием и обработка основной логики выполняются через один сервер автоматизации. Это упрощает сопровождение проекта и снижает риск конфликтов.

### 2. Централизованная логика автоматизации
Основные сценарии и обработка состояний выполняются на стороне iRidi Server, что исключает дублирование логики.

### 3. Гибкая передача данных
В SCADA передаются только необходимые теги, что позволяет не перегружать проект избыточными данными.

### 4. Возможность управления из SCADA
Команды с АРМ диспетчера передаются в SCADA Server, далее — в iRidi Server, который транслирует их на оборудование, обеспечивая согласованность работы.

## Рекомендации по проектированию

При проектировании интеграции важно заранее определить перечень сигналов для диспетчеризации (аварии, параметры климата, статусы связи) и права доступа для различных пользователей. Избегайте попыток прямого подключения к полевым шинам (RS-485, KNX) из двух разных систем — используйте логическую интеграцию по IP-сети между серверами.
