# Руководство по устранению конфликтов глобальных переменных в iRidi Pro

## Обзор
Данная инструкция описывает алгоритм поиска и устранения дубликатов глобальных переменных (ГП) на шине пользователя в среде iRidi Pro. Соблюдение этих рекомендаций предотвращает конфликты адресации и обеспечивает стабильную работу проекта.

## Контекст
Дублирование ГП возникает при некорректной конфигурации устройств или ошибках импорта данных. Это приводит к потере пакетов, неверному срабатыванию команд и нестабильной логике автоматизации. 

## Пошаговая инструкция

### Шаг 1: Подготовка инструментов отладки
Для диагностики системы необходимо активировать расширенные возможности среды:

1. **Обновление устройств:** В интерфейсе iRidi Studio проверьте все устройства. При наличии статуса «Обновить» (синий индикатор) примените его для синхронизации ГП с шиной.
2. **Активация Test Helper:**
   - Создайте ярлык для запуска iRidi Studio.
   - В свойствах ярлыка в поле «Объект» (Target) допишите аргумент: `extension TESTER`.
   - Запустите студию с использованием созданного ярлыка.
3. **Генерация отчета:** В интерфейсе студии откройте вкладку **Test Helper** и нажмите **«Чек дубликат»**. Система экспортирует текстовый файл со списком конфликтующих адресов.

### Шаг 2: Алгоритм устранения ошибок

1. **Анализ данных:** Используйте полученный лог для поиска конкретных адресов ГП в конфигураторе через функцию поиска.
2. **Устранение конфликтов:**
   - **Стандартный случай:** Если дубль обнаружен в двух независимых устройствах, замените ГП одного из них на свободную. Доступность адреса подтвердите через инструмент «Check Reg GV in project».
   - **Связь с контроллером (КУ):** При конфликте в логических связях с КУ проанализируйте карту связей в студии. Идентифицируйте лишнюю связь и удалите её или переназначьте адрес.
3. **Внесение правок:** Перейдите по пути **Девайс → Сабдевайс → Тег**. В поле «Глобальная переменная» введите корректное свободное значение и нажмите кнопку «Изменить».

### Шаг 3: Финализация и верификация

1. **Синхронизация:** После внесения изменений повторно обновите все устройства, чтобы система зафиксировала изменения конфигурации.
2. **Логирование:** Убедитесь в отсутствии ошибок в консоли логов.
3. **Тестирование:** Проверьте работу всех связанных логических сценариев, так как изменение глобальной переменной может потребовать обновления зависимых объектов.

## Рекомендации
Регулярная проверка через Test Helper при добавлении новых устройств — лучшая практика для поддержания здоровья проекта. Всегда создавайте резервную копию проекта перед внесением массовых изменений в структуру ГП.
