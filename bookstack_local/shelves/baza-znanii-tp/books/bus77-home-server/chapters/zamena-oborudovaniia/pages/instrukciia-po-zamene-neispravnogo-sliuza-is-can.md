# Инструкция по замене неисправного шлюза IS-CAN

Данная статья описывает процедуру корректной замены неисправного шлюза IS-CAN в проектах Bus77 Home с сохранением текущей конфигурации устройств и связей.

### Порядок действий

Для корректной замены устройства необходимо выполнить следующие шаги:

1. **Подготовка файлов проекта**:
   - Откройте файлы конфигурации проекта: `Bus77Config.xml`, `Project.bpr` и `RMS.xml`.
   - Найдите параметры `HWID` старого шлюза и замените их на `HWID` нового устройства во всех найденных вхождениях.
   - Замените данные IP-адреса головного устройства линии со старых на данные нового шлюза.

2. **Замена оборудования на объекте**:
   - Снимите старый шлюз с линии.
   - Подключите новый IS-CAN (рекомендуется подключать его в изолированную линию, чтобы избежать конфликтов при автоматическом опросе сети).

3. **Настройка параметров нового устройства**:
   - Через конфигурационный файл пропишите новому шлюзу `Lead Address` (адрес ведущего устройства), идентичный адресу старого шлюза.
   - Установите корректный `User ID`.
   - В тег `DATE&TIME` установите номер ГП (Группового Профиля), который использовался в старом шлюзе (значение можно уточнить в проектной документации или в файлах проекта).

### Важные замечания
- Данная инструкция применима строго для устройств IS-CAN.
- Перед проведением работ убедитесь, что бас не осуществляет активный опрос, чтобы избежать ошибок смены и контроля лид-адресов.
