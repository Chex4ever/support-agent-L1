# Особенности режима Kiosk и управления доступом на панелях iRidi P-серии

## Обзор
Настройка режима «киоск» и управление системным доступом на интерактивных панелях iRidi зависят от поколения оборудования. В данной статье разобраны различия между старым и новым поколениями панелей и методы ограничения интерфейса Android.

## Сравнение поколений

| Характеристика | Панели предыдущего поколения | Новые панели (P8-bb / P10-bb) |
| :--- | :--- | :--- |
| **Системные настройки** | Нет штатной блокировки статус-бара | Есть аппаратная блокировка элементов ОС |
| **Управление доступом** | Только через встроенный функционал стороннего ПО | Системное ограничение доступа для любых приложений |
| **Root-права** | Не отключаются (особенности прошивки) | Оптимизированы для киоск-режима |

## Панели предыдущего поколения
Для панелей (например, с серийными номерами вида `P100001651`) доступ к системным элементам (статус-бар, настройки Android) ограничен программно.

### Ограничения и возможности
- **Стороннее ПО:** Полноценный «киоск» возможен только для приложений, поддерживающих данный режим на уровне собственного кода.
- **Настройка через ADB:** Можно назначить права `Device Owner` приложению, совместимому с режимом киоска, используя команду: `dpm set-device-owner <package_name>/workdata.DeviceAdminHelper`.
- **Важно:** Без встроенной поддержки киоск-режима в самом приложении защитить систему от выхода в настройки или смены домашнего экрана невозможно.

## Работа с i3 Pro
Для фирменного ПО iRidi доступны дополнительные инструменты:
- **Kiosk-версия:** Использование специализированной сборки приложения.
- **Скрипты:** Применение команды `IR.KioskMode(true);`. Этот метод скрывает элементы управления внутри проекта i3 Pro, однако не блокирует доступ к системным процессам Android.

## Новые панели (P8-bb / P10-bb)
Архитектура ОС в новых поколениях была переработана. Система поддерживает аппаратные настройки для принудительного скрытия статус-бара и навигационной панели. Это позволяет перевести любое приложение в режим киоска на системном уровне без необходимости доработки кода самого приложения.

## Рекомендации
1. **Для строгих инсталляций:** Используйте актуальные версии панелей (P8-bb / P10-bb) для гарантированного скрытия элементов управления ОС.
2. **Для старого оборудования:** Учитывайте, что защита интерфейса ограничена возможностями конкретного установленного приложения. Использование стороннего софта без функций киоска не обеспечивает полную безопасность системы.
