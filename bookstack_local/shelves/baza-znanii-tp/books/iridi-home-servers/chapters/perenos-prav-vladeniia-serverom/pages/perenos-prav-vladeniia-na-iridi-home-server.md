# Перенос прав владения на iRidi Home Server

В процессе эксплуатации системы автоматизации может возникнуть необходимость передачи прав владения сервером от аккаунта интегратора к аккаунту конечного пользователя. Для корректного переноса объекта и сохранения всех настроек конфигурации необходимо выполнить процедуру деактивации.

## Предварительные требования
- Наличие доступа к приложению управления сервером (например, I3KNX).
- Нахождение в одной локальной сети с сервером (удаленная деактивация невозможна).
- Знание пароля от текущего аккаунта владельца.

## Пошаговая инструкция по переносу

1. **Деактивация сервера**:
   - Убедитесь, что вы подключены к локальной сети, в которой находится сервер.
   - Откройте приложение, используемое для управления сервером.
   - Перейдите в меню выбора проектов/серверов.
   - Выберите нужный сервер в списке.
   - Выберите опцию «Отключить сервер» (или аналогичную команду деактивации).
   - Введите пароль от текущего аккаунта для подтверждения операции.

2. **Запрос в техническую поддержку**:
   - После успешной деактивации обратитесь в службу поддержки iRidi (support@iridi.com).
   - Предоставьте информацию о текущем аккаунте (владельце) и целевом аккаунте (на который переносится объект).
   - Укажите название объекта для ускорения процесса обработки.

3. **Активация на новом аккаунте**:
   - После того как техническая поддержка подтвердит перенос объекта, войдите в приложение под учетной записью нового владельца.
   - Выполните активацию сервера согласно стандартной процедуре, описанной в документации [Basics of working with i3 KNX](http://dev.iridi.com/Basics_of_working_with_i3_KNX).

> **Важно:** В процессе переноса текущая конфигурация проекта сохраняется на сервере. После активации рекомендуется проверить работоспособность всех систем.
