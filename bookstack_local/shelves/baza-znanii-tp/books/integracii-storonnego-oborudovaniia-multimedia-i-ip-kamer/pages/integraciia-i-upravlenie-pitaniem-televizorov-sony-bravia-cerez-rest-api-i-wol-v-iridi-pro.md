## 1. Метаданные и Экспресс-диагностика
* **Тип обращения:** Интеграция стороннего оборудования / Системное руководство
* **Симптом:** Отсутствие нативного драйвера в iRidi для управления Sony Bravia TV (включение/выключение)
* **Статус объекта:** ПНР / Эксплуатация

## 2. Описание проблемы (Context)
При проектировании и пусконаладке систем автоматизации на базе iRidi Pro интеграторы часто сталкиваются с задачей интеграции телевизоров Sony Bravia. На данный момент готовый нативный драйвер или JS-модуль в комплекте поставки iRidi отсутствует. Управление данными панелями осуществляется посредством отправки сетевых REST-запросов и команд Wake-on-LAN.

Основная техническая сложность заключается в управлении питанием. Выключение панели стабильно выполняется с помощью POST-запроса к API телевизора. Однако, при переходе телевизора в режим глубокого сна (Deep Sleep) его сетевой стек отключается, делая HTTP-управление невозможным. Для гарантированного включения ТВ из любого состояния требуется совмещение REST API запросов и широковещательных пакетов Wake-on-LAN (WOL).

## 3. Diagnostic Mapping (Symptom → Cause)
| Визуальный симптом / Ошибка в логах | Вероятная причина | Обязательные данные для сбора |
| :--- | :--- | :--- |
| Ошибка `HTTP 401 Unauthorized` / `403 Forbidden` | Неверно настроен, изменен или отсутствует заголовок `X-Auth-PSK`, либо ключ авторизации не совпадает с установленным в настройках ТВ. | Скриншот настроек IP-контроля в меню ТВ, дамп HTTP-заголовков из логов iRidi. |
| ТВ выключается по сети, но не реагирует на команды включения по REST API | Телевизор перешел в режим глубокого энергосбережения; отключена сетевая карта. | Проверка статуса опции «Remote Start» / «Network Standby» в меню настроек ТВ. |
| Команда WOL не пробуждает телевизор | Указан неверный MAC-адрес устройства, порт, либо сетевой коммутатор/роутер блокирует прохождение широковещательных пакетов (Broadcast). | Проверка MAC-адреса ТВ по ARP-таблице роутера, топология сети (VLAN). |

## 4. Официальное решение (Root Cause Fix)
Для настройки надежного и стабильного управления питанием выполните следующие действия:

### Шаг 1: Подготовка телевизора Sony Bravia
1. Перейдите в **Settings (Настройки)** -> **Network (Сеть)** -> **Home network setup (Настройка домашней сети)** -> **IP control (Контроль IP)**.
2. Переведите переключатель **Simple IP control (Простое управление IP)** в положение **ON (Вкл)**.
3. В меню **Authentication (Аутентификация)** выберите опцию **Normal and Pre-Shared Key (Нормальный и общий ключ)**.
4. Перейдите в поле **Pre-Shared Key (Общий ключ)** и задайте статический пароль доступа (например, `1234`).
5. Вернитесь в настройки сети, найдите пункт **Remote start (Удаленный запуск)** и установите значение **ON (Вкл)** (это сохранит сетевое подключение активным в Standby-режиме).

### Шаг 2: Настройка в iRidi Pro
1. Добавьте в проект драйвер **AV & Custom Systems (HTTP)**.
2. Задайте статический IP-адрес телевизора.
3. Сформируйте команду выключения (HTTP POST):
   * **URI:** `/sony/system`
   * **Headers:** `Content-Type: application/json`, `X-Auth-PSK: <Ваш_PSK>` (например, `1234`)
   * **Payload (Body):** `{"method":"setPowerStatus","id":1,"params":[{"status":false}],"version":"1.0"}`
4. Для включения используйте отправку пакета Wake-on-LAN на MAC-адрес устройства (порт 9) либо аналогичный POST-запрос со значением `"status":true`.

## 5. Технический Workaround
* **Логика:** Сочетание отправки широковещательного пакета Wake-on-LAN (WOL) для гарантированного пробуждения сетевого интерфейса и программного управления HTTP-драйвером через JS API.
* **Пример кода:**
```javascript
// Пробуждение телевизора по MAC-адресу через Magic Packet (порт 9)
IR.SendWOL("AA:BB:CC:DD:EE:FF", 9);

// Пример динамической сборки payload для отправки в HTTP-драйвер
var payload = JSON.stringify({
    method: "setPowerStatus",
    id: 1,
    params: [{ status: false }],
    version: "1.0"
});

// Отправка JSON-команды выключения в драйвер
// IR.GetDevice("AV & Custom Systems (HTTP)").Send([payload]);
```

В системе iRidi Pro встроенный нативный драйвер для телевизоров Sony Bravia отсутствует, однако интеграция успешно реализуется с использованием универсального драйвера **AV & Custom Systems (HTTP)** и механизма Wake-on-LAN.

Пожалуйста, выполните следующие шаги для настройки:

**1. Настройки на стороне телевизора:**
* Включите **Simple IP control**: Settings -> Network -> Home network setup -> IP control -> Simple IP control -> **ON**.
* Включите авторизацию по ключу: IP control -> Authentication -> **Normal and Pre-Shared Key**.
* Задайте секретный ключ в поле **Pre-Shared Key** (например, `1234`).
* Разрешите удаленный запуск: Settings -> Network -> Remote start -> **ON**.

**2. Управление питанием:**
* **Выключение (HTTP POST):**
  * Эндпоинт: `http://<IP_телевизора>/sony/system`
  * Заголовки:
    `Content-Type: application/json`
    `X-Auth-PSK: <Ваш_PSK_ключ>`
  * Тело запроса (Payload): `{"method":"setPowerStatus","id":1,"params":[{"status":false}],"version":"1.0"}`
* **Включение (WOL / HTTP):**
  * Рекомендуется отправлять Wake-on-LAN (WOL) пакет на физический MAC-адрес телевизора (порт 9). В JS iRidi для этого используется метод: `IR.SendWOL("MAC_АДРЕС", 9);`
  * Если телевизор не успел уйти в глубокий сон, также сработает аналогичный POST-запрос с параметром `{"status":true}`.

## 💡 Ключевые Инсайты (NotebookLM)
1. **Преодоление ограничений Deep Sleep:** Применение только REST API команд не обеспечивает 100% надежности включения ТВ, так как операционная система Android TV при длительном простое полностью отключает сетевой интерфейс. Комбинирование REST API и Wake-on-LAN является единственным технически верным решением.
2. **Упрощенная авторизация через Pre-Shared Key:** Использование механизма `X-Auth-PSK` позволяет полностью обойти сложный процесс OAuth-авторизации и сопряжения устройств, предоставляя прямой доступ к JSON-RPC API телевизора по статическому ключу безопасности.
3. **Влияние сетевой инфраструктуры:** Для корректной работы функции включения через WOL управляющий контроллер iRidi Pro и телевизор должны находиться в одном L2-сегменте сети, либо сетевое оборудование должно поддерживать и иметь активную функцию пересылки направленного широковещания (Directed Broadcast).
