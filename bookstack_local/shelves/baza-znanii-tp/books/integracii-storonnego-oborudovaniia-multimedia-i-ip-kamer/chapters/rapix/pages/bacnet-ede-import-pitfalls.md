# Типичные проблемы импорта BACnet EDE в iRidi Studio

## 1. "Unknown csv file type"

### Причины
- Нет строки `#Engineering-Data-Exchange` в первой строке файла
- Неправильный разделитель (должна быть `;`)
- Неправильное количество колонок (должно быть 16)

### Решение
Добавить magic header, исправить разделитель, удалить лишние колонки.

## 2. Версия layout (VERSION_OF_LAYOUT)

`VERSION_OF_LAYOUT` может быть 2.1 или 2.3 — обе поддерживаются. Версия 2.3 НЕ является причиной отказа в импорте.

## 3. Пустые строки и BOM

- Файл должен быть в UTF-8 (BOM допускается, Studio его игнорирует)
- Пустые строки между данными допускаются
- Строки, начинающиеся с `#`, считаются комментариями

## 4. Кодировка BACnet типов

Studio маппит BACnet объекты на iRidi каналы:
- `AnalogValue` → уровень (0-100%)
- `BinaryValue` → вкл/выкл, сцены
- `AnalogInput` / `BinaryInput` → входы/фидбек
