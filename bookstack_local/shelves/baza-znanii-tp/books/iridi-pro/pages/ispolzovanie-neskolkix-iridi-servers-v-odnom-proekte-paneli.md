# Использование нескольких iRidi Server в одном проекте

## Обзор
Данная статья описывает методы масштабирования систем автоматизации путем интеграции нескольких контроллеров iRidi Server в единый интерфейс управления. Этот подход позволяет оптимизировать вычислительную нагрузку на крупных объектах, сохраняя при этом целостный пользовательский опыт.

## Контекст
При автоматизации масштабных объектов (например, многоэтажных зданий) рационально разделять оборудование на логические зоны. Использование нескольких серверов позволяет распределить нагрузку по локальным шинам (KNX, Modbus, AV), превращая панель управления в единую точку доступа к независимым серверным узлам.

## Реализация

### Настройка подключений
Панель управления поддерживает одновременное активное соединение с несколькими инстансами iRidi Server. 
1. В проекте панели в разделе **Connections** необходимо добавить отдельные узлы iRidi Server для каждого физического контроллера.
2. Укажите соответствующие IP-адреса для каждого соединения.
3. Интегрируйте элементы управления из всех серверов в единую визуальную среду.

### Архитектурные особенности
iRidi Server не выполняет автоматическую репликацию данных. При проектировании учитывайте:
* **Изоляция логики:** Сцены, таймеры и глобальные переменные должны быть настроены внутри каждого сервера индивидуально.
* **Именование:** Используйте префиксы зон (например, `F1_`, `F2_`) для тегов, чтобы исключить конфликты имен при импорте в проект панели.

## Рекомендации по интеграции
Для обеспечения стабильного взаимодействия между сегментами системы используйте следующие подходы:

1. **Глобальные команды:** Панель управления может выступать «оркестратором», отправляя широковещательные команды (например, «Выключить всё») на все серверы параллельно.
2. **Межсерверный обмен:** Для передачи данных между серверами используйте протоколы MQTT, HTTP API или Modbus.
3. **Master-Slave:** Назначьте один сервер ответственным за выполнение глобальных сценариев, передавая команды на периферийные серверы через API.

## Заключение
Использование многосерверной архитектуры — стандарт для сложных объектов. Грамотное планирование связей и строгая модульность проектов серверов обеспечивают высокую отказоустойчивость и упрощают последующее обслуживание системы.
