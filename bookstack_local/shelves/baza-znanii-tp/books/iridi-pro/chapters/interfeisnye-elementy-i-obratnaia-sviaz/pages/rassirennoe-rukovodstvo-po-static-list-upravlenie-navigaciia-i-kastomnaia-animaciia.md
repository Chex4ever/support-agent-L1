# Руководство по работе с Static List: API, навигация и кастомная анимация

## 1. Введение
При разработке сложных интерфейсов в iRidi Pro Studio стандартный компонент `Static List` часто требует расширения функциональности. Данное руководство описывает текущие ограничения API и предлагает альтернативные подходы для реализации продвинутой навигации.

## 2. Управление Static List: нюансы API
Событие `IR.EVENT_LIST_SCROLL` является основным инструментом отслеживания прокрутки, однако имеет ряд критических особенностей:

* **Триггер события:** Работает только при типе прокрутки *Center*.
* **Ограничения:** Событие не генерируется при недостаточной амплитуде жеста или при попытке прокрутки за границы списка.
* **Проблема EDGE:** Наблюдается некорректная работа свойства `EDGE`. Ошибка находится в статусе обработки.

### Рекомендации по отслеживанию позиции
Нативный метод `list.GetPosition()` находится в разработке (тикет #572-567564). В качестве временного решения используйте:
- Ручной учет позиции через глобальные переменные.
- Токены, обновляемые внутри обработчика `IR.EVENT_LIST_SCROLL`.

## 3. Продвинутые UX: реализация кастомных свайп-эффектов
Если стандартный список не обеспечивает нужную плавность, рекомендуется ручная реализация свайпа через контейнеры и события `onTouch`.

### Принципы реализации
Используйте один полноэкранный попап с дочерними контейнерами, управляя их координатами (`x/y`) и параметром `opacity` для достижения эффекта анимации.

```javascript
var startX = 0, 
    screenWidth = IR.GetVariable("System.W"),
    currentPopup = IR.GetItem("Main.PopupCurrent"), 
    nextPopup = IR.GetItem("Main.PopupNext"),
    swipeStarted = false;

function onTouchDown(x) { startX = x; swipeStarted = true; }

function onTouchMove(x) {
    if (!swipeStarted) return;
    var deltaX = x - startX;
    currentPopup.x = Math.min(0, deltaX);
    nextPopup.x = screenWidth + deltaX;
    nextPopup.opacity = Math.abs(deltaX) / screenWidth;
}

function onTouchUp(deltaX) {
    swipeStarted = false;
    Math.abs(deltaX) > screenWidth * 0.3 ? completeSwipe() : rollbackSwipe();
}
```

## 4. Перспективное развитие GUI
На основе запросов сообщества в разработке находятся следующие возможности:
* **Управление Z-order:** Программное изменение слоев элементов.
* **Вращение:** Нативное управление поворотом графических элементов через скрипты.
* **Мультимедиа и Scale:** Поддержка MP4, GIF и настройка точки масштабирования (Scale Center).
