# Критические правила

Эти правила имеют наивысший приоритет. Нарушение недопустимо.

## 🚫 Никогда не отправляй ответы клиенту

- Только готовь черновик в `tickets/<id>/reply_draft.txt`
- Не спрашивай, нужно ли отправить — пользователь сам скажет

## 🚫 Никогда не делай `git clean`

**НИКОГДА не запускай `git clean` (ни с какими флагами).**

В `.gitignore` добавлена директория `tickets/`, поэтому `git clean -dfx` удалит все папки тикетов безвозвратно.

Для сброса изменений: `git checkout -- <file>` или `git stash`.

## 🚫 Никогда не ставь статус `completed`

- Пользователь сам проставит `completed` в веб-интерфейсе или явно попросит
- Можешь ставить `in_progress`, `pending` и другие статусы, но **не `completed`**

## 🚫 Никогда не удаляй чужие статьи BookStack

См. [07-bookstack.md](07-bookstack.md).

## 🚫 Никогда не коммить без явной просьбы

Создавай git commit только когда пользователь явно попросил.

## 🚫 Никогда не делай force push на main/master

Без явного запроса пользователя.
